# mystory-blog Implementation Playbook (Code + UI/UX + SEO)

Date: 2026-02-10
Scope: Consolidated implementation plan based on repository code review, UI/UX review, and SEO investigation.

## 1. Objective

Ship a stable, searchable, and discoverable personal blog that supports both technical content and everyday stories while preserving the current Astro static architecture.

Primary outcomes:
- No known metadata correctness bugs.
- Accessibility baseline fixed for key user journeys.
- SEO metadata and structured data coverage expanded beyond post pages.
- Deterministic builds and repeatable QA checks.

## 2. Current Baseline (Evidence Snapshot)

### Build and quality checks
- `npm run check`: pass.
- `npm run build`: pass.
- `npm run test:e2e`: pass (after Playwright browser install).
- `npm run lint`: fail due to unused variable in `scripts/prepare_hero_photo.mjs`.

### Accessibility and UX findings
- Multi-page axe scan found:
  - `color-contrast` issues on blog tag chips.
  - `label-title-only` issue on search input.

### SEO findings
- Broken Open Graph image on `/about`: `og:image` points to `/og/about.png` but file does not exist.
- RSS items are not sorted by date descending.
- Missing `hreflang` alternates across all generated HTML pages.
- JSON-LD exists for home and posts, but many index/discovery pages have none.

## 3. Implementation Principles

- Keep changes incremental and reviewable.
- Fix correctness issues before enhancements.
- Preserve static generation and avoid runtime SEO dependencies.
- Add measurable acceptance criteria for each task.

## 4. Priority Roadmap

## Phase 0: Correctness and Determinism (High Priority)

### 0.1 Fix `/about` metadata and schema context
Problem:
- `/about` currently reuses blog post layout, generating blog-specific breadcrumb and non-existent OG image path.

Implementation:
- Create a dedicated `About` page layout or make `BlogPost.astro` behavior configurable for non-blog pages.
- Ensure `/about` uses an existing OG image (hero or placeholder) and valid schema type (`AboutPage` or `ProfilePage` + `Person`), not forced `BlogPosting`.
- Remove blog breadcrumb dependency for `/about`.

Files:
- `src/pages/about.astro`
- `src/layouts/BlogPost.astro` (if shared behavior stays)
- `src/utils/jsonld.ts`

Acceptance criteria:
- `/about` output has valid canonical and OG tags.
- `og:image` URL resolves to an existing file.
- JSON-LD on `/about` does not claim post semantics.

Validation:
- `npm run build`
- Verify `dist/about/index.html` meta tags.
- Verify referenced OG image exists in `dist`.

### 0.2 Sort RSS items by publish date descending
Problem:
- Feed ordering is inconsistent, reducing feed quality for subscribers.

Implementation:
- Sort collection entries by `pubDate` descending before mapping to RSS items.

Files:
- `src/pages/rss.xml.js`

Acceptance criteria:
- Newest posts appear first in `dist/rss.xml`.

Validation:
- `npm run build`
- Inspect first N items in `dist/rss.xml` for descending chronology.

### 0.3 Fix lint blocker
Problem:
- Lint fails on one unused catch variable.

Implementation:
- Remove or use the unused variable in `scripts/prepare_hero_photo.mjs`.

Files:
- `scripts/prepare_hero_photo.mjs`

Acceptance criteria:
- `npm run lint` passes.

### 0.4 Remove randomized fallback images on blog index
Problem:
- `Math.random()` in static build causes unstable visuals and cache churn.

Implementation:
- Replace random fallback with deterministic mapping, for example:
  - hash of `post.id` modulo placeholder array length, or
  - fixed default image.

Files:
- `src/pages/blog/index.astro`

Acceptance criteria:
- Repeated builds produce same fallback image per post.

Validation:
- Run build twice and compare blog index image assignments.

## Phase 1: Accessibility and Core UX (High Priority)

### 1.1 Fix blog tag chip color contrast
Problem:
- Tag pills fail WCAG contrast in current palette.

Implementation:
- Darken text and/or adjust chip background alpha per theme.
- Validate in both light and dark modes.

Files:
- `src/pages/blog/index.astro`
- optionally `src/styles/global.css` (if extracting tokenized color variables)

Acceptance criteria:
- Axe reports zero `color-contrast` violations on `/blog`.

### 1.2 Add visible label for search input
Problem:
- Search input is generated by Pagefind UI without visible label context.

Implementation options:
- Add an explicit visible label wrapper around the search mount area.
- Post-initialize script hook to associate `<label for=...>` with generated input.
- Ensure label remains visible on all breakpoints.

Files:
- `src/pages/search.astro`
- `src/styles/search.css`

Acceptance criteria:
- Axe reports zero `label-title-only` on `/search`.

### 1.3 Add skip link for keyboard navigation
Problem:
- Repeated header navigation increases keyboard friction.

Implementation:
- Add skip link in page/layout root and visible-on-focus styling.
- Target `main` content container IDs consistently.

Files:
- shared layouts/pages using header + main (start with `src/layouts/BlogPost.astro` and top-level pages)
- `src/styles/global.css`

Acceptance criteria:
- First tab focuses skip link and jumps to main content.

### 1.4 Improve mixed-content IA for tech + life stories
Problem:
- Blog list is one stream; intent-based scanning is weak.

Implementation:
- Introduce top-level content sections (for example: `Tech`, `Life`).
- Add filter tabs or segmented list on `/blog`.
- Reuse tags/categories and/or directory-based taxonomy.

Files:
- `src/pages/blog/index.astro`
- `src/utils/tags.ts` (if taxonomy helper changes)

Acceptance criteria:
- Reader can isolate technical vs personal posts in one interaction.

## Phase 2: SEO Coverage and Discoverability (High Priority)

### 2.1 Add `hreflang` alternate strategy
Problem:
- No language alternate links exist, limiting multilingual SEO clarity.

Implementation:
- Extend `BaseHead` props to accept optional alternates.
- Emit `link rel="alternate" hreflang="..." href="..."` when counterpart exists.
- Include `x-default` where appropriate.

Files:
- `src/components/BaseHead.astro`
- route-level pages where alternates can be computed

Acceptance criteria:
- Pages with language variants include valid `hreflang` clusters.

### 2.2 Expand structured data coverage for index pages
Problem:
- Many route pages lack JSON-LD entirely.

Implementation:
- Add route-appropriate schema:
  - `/blog`: `CollectionPage` + `ItemList`
  - `/tags`: `CollectionPage`
  - `/tags/[tag]`: `CollectionPage` + `ItemList`
  - `/search`: `WebPage`
  - `/cat-pics`: `CollectionPage`/`ImageGallery`
- Keep existing post `BlogPosting` schema intact.

Files:
- `src/pages/blog/index.astro`
- `src/pages/tags/index.astro`
- `src/pages/tags/[tag].astro`
- `src/pages/search.astro`
- `src/pages/cat-pics.astro`
- `src/utils/jsonld.ts`

Acceptance criteria:
- Each major index/discovery page emits one valid JSON-LD block.

### 2.3 Scope Pagefind indexing to main content
Problem:
- Full-body indexing can dilute relevance with navigation/footer text.

Implementation:
- Add `data-pagefind-body` to page main content containers.
- Optionally exclude utility regions from indexing.

Files:
- layout/page templates containing content bodies

Acceptance criteria:
- Build output no longer warns about missing `data-pagefind-body`.
- Search results are more content-centric.

### 2.4 Remove OG generation network dependency risk
Problem:
- OG build currently fetches fonts from external API during build.

Implementation:
- Bundle and configure local font files for `astro-og-canvas`.
- Ensure consistent rendering in CI/offline scenarios.

Files:
- `src/pages/og/[...slug].ts`
- `public/fonts/` or project font asset location

Acceptance criteria:
- OG build does not require external font fetch.

## Phase 3: Content and Localization Quality (Medium Priority)

### 3.1 Normalize language metadata in content frontmatter
Problem:
- Most posts omit `lang`, reducing locale precision for metadata and cards.

Implementation:
- Backfill `lang` for existing posts.
- Add content lint/check script for required language declaration policy.

Files:
- `src/content/blog/**/*.md*`
- optional content QA script

Acceptance criteria:
- All posts have explicit `lang`.

### 3.2 Improve date localization UX
Problem:
- `FormattedDate` is fixed to `en-us` even on Korean pages.

Implementation:
- Format based on page/post language.
- Pass locale explicitly from layout/page.

Files:
- `src/components/FormattedDate.astro`
- calling pages/layouts

Acceptance criteria:
- Korean pages show Korean-style dates; English pages show English-style dates.

## 7. Delivery Criteria for Final Sign-off

- No broken OG URLs in generated pages.
- RSS chronology correct and stable.
- Lint/type/build/e2e all green.
- Axe: zero critical issues on core routes.
- Route-level schema coverage complete for home, blog index, post, tags, search, about, and cat gallery.
- Search index scoped to content body and relevance manually spot-checked.

## 8. Non-goals

- Full visual redesign.
- Migration away from Astro static architecture.
- Replacing Giscus or Pagefind.
